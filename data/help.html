Layout and Interaction

The left pane contains the always-visible search box, module list, search results, tags, and the preview pane at the bottom. The center pane shows Bible text for the active study tab. The right pane shows commentary, general books, dictionaries, and editable documents.

Single-clicking a search result updates the preview pane. Double-clicking a result opens it in the active study tab. Middle-clicking opens it in a new study tab. Hovering a Bible word can populate the lower-left MAG preview with Strong's and morphology details.

Verse numbers are clickable in the Bible and commentary panes. Commentary and general-book scripture links open the referenced verse. Internal table-of-contents links scroll within the current document.

Search Basics

The Search tab module chooser controls which Bible text is searched. When a search starts from a Bible word context menu, Verdad automatically uses the module for the word that was clicked.

Multi-word finds verses containing the search words, regardless of order. Example: faith hope charity.

Exact phrase matches the words together in order. Example: in the beginning.

Strong's / lemma searches indexed Strong's data. You can enter G3588, H7225, strongs:G3588, or lemma:H7225. Right-clicking a Bible word also exposes Strong's search actions.

Search results are sorted canonically by book, chapter, and verse. The preview pane shows the rendered verse, including working note and cross-reference markers when the source module provides them.

Regex Search

Regex search uses ECMAScript syntax and is case-insensitive. It runs against the indexed plain-text Bible content. If a module has not finished indexing yet, regex search waits for that index before it can return matches.

Useful building blocks:

. matches any single character except a newline.

*, +, and ? repeat the previous token. Examples: grac(e|es), king(dom)?.

[abc] matches one character from a set. [^abc] matches one character not in the set.

(a|b) creates alternatives. Example: \b(faith|hope|charity)\b.

^ anchors the pattern to the start of the verse text. $ anchors to the end.

\b matches a word boundary. This is the safest way to avoid partial-word matches. Example: \bson\b will not match sons.

\s matches whitespace and \w matches word characters. When you need punctuation literally, escape it: \?, \., \(.

Practical regex examples:

^In\b finds verses beginning with “In”.

\b(king|kingdom)\b finds either whole word.

God.*spirit finds verses where “God” appears before “spirit” with anything between them.

\b[aA]men\b matches “Amen” or “amen” explicitly, though the search is already case-insensitive.

If a regex produces too many results, narrow it with anchors, explicit word boundaries, or more precise alternatives.

Study Tools

The Bible toolbar lets you switch modules, move between chapters, jump to a reference, toggle paragraph mode, and enable parallel columns. Additional toggle buttons control whether inline study markers are shown for Strong's data, morphology, note markers, and cross references.

Right-clicking a Bible word opens the study context menu. From there you can copy the word, search for the word, search Strong's numbers behind the word, look the word up in a dictionary, or add a tag for the current verse.

The dictionary pane at the bottom of the right side is always available for lexicon and dictionary lookups. It follows Strong's and plain-word lookups without leaving the current study tab.

General Books

The General Books tab now uses the module's table of contents directly. The first chooser selects the module and the second chooser selects a TOC entry. When you choose an entry that has children, Verdad renders that entry and its descendants together so you can scroll continuously through the section.

The page also includes an inline contents list that jumps to sections already loaded in the current view. Scripture references inside general books open the referenced verse, and note links can populate the preview pane when the module exposes note data.

Tags

Tags are verse-based and stored separately from the search index. Open the Tags tab in the left pane to browse tags, filter them, rename them, remove verses, and preview tagged verses.

Tagged verses in the Bible pane now end with a single marker instead of one marker per tag. Hover the marker to see every tag attached to that verse, and click it to open the tag list for that verse.

Practical Tips

If a search result or verse preview shows note markers, click them in the preview pane to read the footnote or cross-reference content without losing your place in the result list.

If you want to compare translations, enable parallel view and keep the first module as the “master” module for searching and navigation.

If the right pane feels crowded, resize the splitters. Verdad remembers pane sizes, open study tabs, and the active document between sessions.
