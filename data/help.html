Layout and Interaction

The left pane contains the always-visible search box, the Modules/Search/Tags tabs, and the preview pane at the bottom. The center pane shows Bible text for the active study tab. The right pane shows commentary, general books, documents, and the always-available dictionary pane at the bottom.

The preview pane is shared by search previews, module metadata, MAG hover output, and link previews from the Bible, commentary, and general-book panes. In the Modules tab, selecting a module previews its metadata before you activate it.

Single-clicking a search result updates the preview pane. Double-clicking a result opens it in the active study tab. Middle-clicking opens it in a new study tab. Verse numbers are clickable in the Bible and commentary panes, and internal links either scroll in place, update the preview, or populate the Search tab with linked verses.

Study Tabs and Session Restore

Each study tab keeps its own Bible module, reference, paragraph or parallel mode, parallel module list, Bible scroll position, commentary module and scroll position, dictionary lookup, general-book selection, and splitter sizes. Use the + button or Ctrl+T to duplicate the current study tab.

When Verdad closes, it saves the main window size and position, the left preview height, the active study tab, all restored study tabs, and the currently opened document path. The Documents tab is global, not per-study-tab, so every study tab shares the same open document.

Search Basics

The Search tab module chooser controls which Bible is searched. When a search starts from a Bible word context menu, Verdad automatically uses the module for the word that was clicked.

Multi-word finds verses containing the search words in any order. Example: faith hope charity.

Exact phrase matches the words together in order. Example: in the beginning.

Regex searches the indexed plain-text Bible content. Regex search waits for the selected module to finish indexing before it can return matches.

Strong's / lemma searches indexed Strong's data. You can enter G3588, H7225, strongs:G3588, or lemma:H7225. Right-clicking a Bible word also exposes Strong's search and dictionary actions.

Search results are sorted canonically by book, chapter, and verse. Single-click a result to preview it, double-click it to navigate the current study tab, or middle-click it to open the verse in a new study tab.

Regex Search

Regex search uses ECMAScript syntax and is case-insensitive. It runs against the indexed plain-text Bible content rather than the rendered XHTML.

Useful building blocks:

. matches any single character except a newline.

*, +, and ? repeat the previous token. Examples: grac(e|es), king(dom)?.

[abc] matches one character from a set. [^abc] matches one character not in the set.

(a|b) creates alternatives. Example: \b(faith|hope|charity)\b.

^ anchors the pattern to the start of the verse text. $ anchors to the end.

\b matches a word boundary. This is the safest way to avoid partial-word matches. Example: \bson\b will not match sons.

\s matches whitespace and \w matches word characters. When you need punctuation literally, escape it: \?, \., \(.

Practical regex examples:

^In\b finds verses beginning with “In”.

\b(king|kingdom)\b finds either whole word.

God.*spirit finds verses where “God” appears before “spirit” with anything between them.

\b[aA]men\b matches “Amen” or “amen” explicitly, though the search is already case-insensitive.

If a regex produces too many results, narrow it with anchors, explicit word boundaries, or more precise alternatives.

Study Tools

The Bible toolbar lets you switch modules, move between chapters, jump to a reference, toggle paragraph mode, and enable parallel columns. Parallel view supports up to seven Bible columns, and the first column remains the master module for navigation and search.

Additional toggle buttons control whether inline study markers are shown for Strong's data, morphology, footnotes, and cross references. Hiding those markers does not disable hover lookups or the Bible word context menu.

Hovering a Bible word can populate the lower-left MAG preview after the configured delay. Right-clicking a word opens the study context menu, where you can copy the word or verse reference, search for the word, search Strong's numbers behind the word, look the word up in a dictionary, or add a tag for the current verse.

Commentary, Dictionary, and General Books

The commentary tab follows the selected Bible verse. Clicking a commentary gutter verse number syncs the Bible pane to that verse. Commentary links can open linked verses in the Search tab or preview related content in the lower-left preview pane.

The dictionary pane at the bottom of the right side is always available for lexicon and dictionary lookups. It follows Strong's and plain-word lookups without leaving the current study tab, and its default modules come from the Settings dialog.

The General Books tab uses the module's table of contents directly. The first chooser selects the module and the second chooser selects a TOC entry. When you choose an entry that has children, Verdad renders that entry and its descendants together so you can scroll continuously through the section.

The page also includes an inline contents list that jumps to sections already loaded in the current view. Scripture references inside general books can populate the Search tab with linked verses, while note links and single references can update the preview pane.

Documents and Editable Commentary

The Documents tab is a lightweight HTML note editor. You can create, open, save, and close HTML files from the File menu or from the buttons inside the tab. The editor supports bold, italic, lists, text sizing, and horizontal rules.

Verse references typed into a document are turned into clickable links in the saved HTML. Verdad restores the last opened document path when the session is restored.

The Commentary tab also exposes an Edit button when the current module is writable. Editing changes the raw entry for the current commentary reference and writes it back through SWORD when you save.

Tags

Tags are verse-based and stored separately from the search index. Open the Tags tab in the left pane to browse tags, filter them by tag name or verse reference, rename them, delete them, remove verses from them, and preview tagged verses.

Tagged verses in the Bible pane end with a single marker instead of one marker per tag. Hover the marker to see every tag attached to that verse, and click it to open the tag list for that verse.

Inside the Tags tab, double-clicking a verse opens it in the active study tab. Middle-clicking opens it in a new study tab.

Modules and Settings

File > Module Manager... opens the built-in SWORD module manager. It can add local sources, add remote sources, refresh source content, and install or update modules. Remote sources can reveal network activity, so use local sources when that matters for your situation.

View > Settings... controls the application font, text font, text size, hover delay, default Greek and Hebrew Strong's dictionaries, and language-specific word dictionaries. F1 opens this help window.

Practical Tips

Ctrl+G opens Go to Verse, Ctrl+P toggles parallel Bibles, Ctrl+T duplicates the current study tab, Ctrl+N/O/S/W control the Documents tab, and Ctrl+Q quits.

If a search result or verse preview shows note markers, click them in the preview pane to read the footnote or cross-reference content without losing your place in the result list.

If you want to compare translations, enable parallel view and keep the first module as the master module for searching and navigation. If the right pane feels crowded, resize the splitters. Verdad remembers pane sizes, study tabs, and the active document path between sessions.
