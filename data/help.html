Layout and Interaction

The left pane contains the always-visible search box, the Modules/Search/Tags tabs, and the preview pane at the bottom. The center pane shows Bible text for the active study tab. The right pane shows commentary, general books, Studypad, and the always-available dictionary pane at the bottom.

The preview pane is shared by search previews, module metadata, hover previews from the Bible pane, and link previews from the Bible, commentary, and general-book panes. In the Modules tab, selecting a module previews its metadata before you activate it.

Single-clicking a search result updates the preview pane. Double-clicking a result opens it in the active study tab. Middle-clicking opens it in a new study tab. Verse numbers are clickable in the Bible and commentary panes, and internal links either scroll in place, update the preview, or populate the Search tab with linked verses.

Study Tabs and Session Restore

Each study tab keeps its own Bible module, reference, paragraph or parallel mode, parallel module list, Bible scroll position, commentary module and scroll position, dictionary lookup, general-book selection, and splitter sizes. Use the + button or Ctrl+T to duplicate the current study tab.

When Verdad closes, it saves the main window size and position, the left preview height, the active study tab, all restored study tabs, and the currently opened studypad path. The Studypad tab is global, not per-study-tab, so every study tab shares the same open studypad.

Search Basics

The Search tab module chooser controls which Bible is searched. When a search starts from a Bible word context menu, Verdad automatically uses the module for the word that was clicked.

Multi-word finds verses containing the search words in any order. Example: faith hope charity.

Exact phrase matches the words together in order. Example: in the beginning.

Regex searches the indexed plain-text Bible content. Regex search waits for the selected module to finish indexing before it can return matches.

Strong's / lemma searches indexed Strong's data. You can enter G3588, H7225, strongs:G3588, or lemma:H7225. Right-clicking a Bible word also exposes Strong's search and dictionary actions.

Search results are sorted canonically by book, chapter, and verse. Single-click a result to preview it, double-click it to navigate the current study tab, or middle-click it to open the verse in a new study tab.

Regex Search

Regex search uses ECMAScript syntax and is case-insensitive. It runs against the indexed plain-text Bible content rather than the rendered XHTML.

Useful building blocks:

. matches any single character except a newline.

*, +, and ? repeat the previous token. Examples: grac(e|es), king(dom)?.

[abc] matches one character from a set. [^abc] matches one character not in the set.

(a|b) creates alternatives. Example: \b(faith|hope|charity)\b.

^ anchors the pattern to the start of the verse text. $ anchors to the end.

\b matches a word boundary. This is the safest way to avoid partial-word matches. Example: \bson\b will not match sons.

\s matches whitespace and \w matches word characters. When you need punctuation literally, escape it: \?, \., \(.

Practical regex examples:

^In\b finds verses beginning with “In”.

\b(king|kingdom)\b finds either whole word.

God.*spirit finds verses where “God” appears before “spirit” with anything between them.

\b[aA]men\b matches “Amen” or “amen” explicitly, though the search is already case-insensitive.

If a regex produces too many results, narrow it with anchors, explicit word boundaries, or more precise alternatives.

Study Tools

The Bible toolbar lets you switch modules, move between chapters, jump to a reference, toggle paragraph mode, and enable parallel columns. Parallel view supports up to seven Bible columns, and the first column remains the master module for navigation and search.

Additional toggle buttons control whether inline study markers are shown for Strong's data, morphology, footnotes, and cross references. Hiding those markers does not disable hover lookups or the Bible word context menu.

Hovering a Bible word can populate the lower-left preview pane after the configured delay. Right-clicking without a text selection opens the study context menu for the clicked verse, where you can copy the full verse with the reference placed first or last, search for the word, search Strong's numbers behind the word, look the word up in a dictionary, or add a tag for the current verse.

When text is selected in the Bible pane, the context menu switches to selection-based copy items instead of word and Strong's actions. Single-verse partial selections copy whole words and add ellipses when the selection omits part of the verse, whole-verse selections copy the verse text without the verse number, and multi-verse selections copy every touched verse using the current verse-per-line or paragraph layout while keeping verse numbers in the copied text. Each copy action is available in both Ref first and Ref last forms.

Commentary, Dictionary, and General Books

The commentary tab follows the selected Bible verse. Clicking a commentary gutter verse number syncs the Bible pane to that verse. Commentary links can open linked verses in the Search tab or preview related content in the lower-left preview pane.

The dictionary pane at the bottom of the right side is always available for lexicon and dictionary lookups. It follows Strong's and plain-word lookups without leaving the current study tab, and its default modules come from the Settings dialog.

The General Books tab uses the module chooser plus previous/next navigation buttons for table of contents movement. The Contents button opens a flyout tree so you can jump directly to any section without keeping a large dropdown open.

Verdad loads general books a page at a time from the selected table of contents entry instead of extending the document while you scroll. Very short sections can pull in a few following entries to fill the page, while large sections stay bounded to the current selection. Scripture references inside general books can populate the Search tab with linked verses, while note links and single references can update the preview pane.

Studypad and Editable Commentary

The Studypad tab is a lightweight HTML note editor. Studypads are stored in Verdad's config directory, for example ~/.config/verdad/studypad on Linux, using the .studypad extension. The dropdown at the top of the tab lists available studypads without the extension, and the toolbar buttons create a new studypad, save the current one, export it to OpenDocument Text (.odt) through LibreOffice when it is available in your PATH, or delete the current file after a confirmation warning. The save button is enabled only when the current studypad has unsaved changes. The editor supports bold, italic, lists, text sizing, and horizontal rules.

Saving a new studypad asks only for a name and writes it into the Studypad directory automatically. Verse references typed into a studypad are turned into clickable links in the saved content. Verdad restores the last opened studypad path when the session is restored.

The Studypad editor has a right-click menu for selected text with copy, cut, and paste. Right-clicking a saved verse-reference link can also insert the linked verse text before or after that link using the current study tab's Bible module. Inserted verse text preserves basic inline styling such as bold and italics, omits Strong's, morphology, notes, and cross-reference markers, and inserts multiple linked verses with verse numbers and line breaks between them.

The Commentary tab also exposes an Edit button when the current module is writable. Editing changes the raw entry for the current commentary reference and writes it back through SWORD when you save.

Tags

Tags are verse-based and stored separately from the search index. Open the Tags tab in the left pane to browse tags, filter them by tag name or verse reference, rename them, delete them, remove verses from them, and preview tagged verses.

Tagged verses in the Bible pane end with a single marker instead of one marker per tag. Hover the marker to see every tag attached to that verse, and click it to open the tag list for that verse.

Inside the Tags tab, double-clicking a verse opens it in the active study tab. Middle-clicking opens it in a new study tab.

Modules and Settings

Tools > Module Manager... opens the built-in SWORD module manager. It can add local sources, add remote sources, refresh source content, and install or update modules. Remote sources can reveal network activity, so use local sources when that matters for your situation.

Tools > Settings... controls the application font, text font, text size, hover delay, editor tab width, default Greek and Hebrew Strong's dictionaries, and language-specific word dictionaries. Tools > Export Settings... writes your preferences, tags, and studypads to verdad_settings.zip, and Tools > Import Settings... restores them without changing the current window and pane layout. F1 opens this help window.

Practical Tips

Ctrl+P toggles parallel Bibles, Ctrl+T duplicates the current study tab, Ctrl+N creates a new studypad, Ctrl+S saves the current studypad, and Ctrl+Q quits.

If a search result or verse preview shows note markers, click them in the preview pane to read the footnote or cross-reference content without losing your place in the result list.

Inside the Studypad editor, Enter continues the current indent or list item, and Tab/Shift+Tab indent or outdent one tab level using the configured editor tab width.

If you want to compare translations, enable parallel view and keep the first module as the master module for searching and navigation. If the right pane feels crowded, resize the splitters. Verdad remembers pane sizes, study tabs, and the active studypad path between sessions.
