• BibleTime does this in the render pipeline, not with a later “strip pass” in the UI:

  1. It installs custom SWORD render filters per module markup type (OSIS/ThML/GBF).
     cswordbackend.cpp (/home/moses/Projects/bible/bibletime/src/backend/managers/cswordbackend.cpp:172)
  2. Those filters convert Strong’s/morph markup into HTML attributes on word spans, and suppress visible Strong’s markup text.
     osistohtml.cpp (/home/moses/Projects/bible/bibletime/src/backend/filters/osistohtml.cpp:87)
     osistohtml.cpp (/home/moses/Projects/bible/bibletime/src/backend/filters/osistohtml.cpp:247)
     gbftohtml.cpp (/home/moses/Projects/bible/bibletime/src/backend/filters/gbftohtml.cpp:74)
     thmltohtml.cpp (/home/moses/Projects/bible/bibletime/src/backend/filters/thmltohtml.cpp:45)
  3. The read display injects JS into the HTML view, and JS reports hovered DOM attributes (like lemma/morph) after a delay.
     bthtmlreaddisplay.cpp (/home/moses/Projects/bible/bibletime/src/frontend/display/bthtmlreaddisplay.cpp:205)
     bthtml.js (/home/moses/Projects/bible/bibletime/src/frontend/display/bthtml.js:71)
  4. C++ receives those attributes and builds Mag info items from lemma/morph attributes (not from visible text).
     bthtmljsobject.cpp (/home/moses/Projects/bible/bibletime/src/frontend/display/bthtmljsobject.cpp:116)
  5. The Mag/Info pane resolves Strong’s and morph definitions by looking up configured lexicon modules (standardGreekStrongsLexicon, standardHebrewStrongsLexicon, etc.) and rendering those entries.
     cinfodisplay.cpp (/home/moses/Projects/bible/bibletime/src/frontend/cinfodisplay.cpp:368)
     cinfodisplay.cpp (/home/moses/Projects/bible/bibletime/src/frontend/cinfodisplay.cpp:404)
  6. Global filter options are still managed via SWORD options (Strong’s Numbers, Morphological Tags, Lemmas).
     cswordbackend.cpp (/home/moses/Projects/bible/bibletime/src/backend/managers/cswordbackend.cpp:244)
     btconfig.cpp (/home/moses/Projects/bible/bibletime/src/backend/config/btconfig.cpp:173)

  So the visible Bible text is clean, while hover definitions come from hidden per-word attributes embedded during filtering.
